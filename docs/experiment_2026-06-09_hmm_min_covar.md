# 실험: HMM 공분산 floor (min_covar) — 사후확률 포화 완화 시도

> **요약**: ① HMM 사후확률이 0/100으로 포화돼 휘프소가 발생한다는 가설 하에 min_covar를 0.001~0.50으로 스윕해 포화 완화 효과를 검증했다. ② min_covar를 올려도 리밸 횟수가 오히려 증가(542→545~546)하고 Sharpe(0.84→0.79~0.82)·Bear22 낙폭(-8.5%→-8.9%)이 악화됐으며, blend EWMA·RF 블렌딩·confidence_smoothing 등이 이미 포화를 흡수하고 있어 더 흐릿하게 만들면 결정력만 잃는다는 사실을 확인했다. ③ baseline 0.001 유지 — 처방이 가설대로 작동하지 않았으며, 최근 휘프소 완화는 이미 적용·검증된 confidence_smoothing이 담당한다.

- 날짜: 2026-06-09
- 코드: `HmmRegimeClassifier(min_covar=...)` (trading/regime.py), run.py·engine.py 배선, config `hmm.min_covar`
- 스윕: `scripts/sweep_hmm_min_covar.py`
- 결론: **효과 없음 → 채택 안 함. 라이브 baseline 0.001 유지.**

## 동기 (진단)

2026-06-08~09 사이 HMM이 단독으로 Slowdown 100%로 하드 플립하며 blend를 방어 쪽으로
끌어내렸다. 그런데 같은 시점 거시·펀더멘털은 전부 위험선호였다:
모멘텀 3M +9.5%, HY 스프레드 z -1.07(타이트), 10Y-2Y +0.41(정상), NFCI -0.49(완화),
VIX 18.9. 규칙기반·RF 모두 Goldilocks(RF 81%). HMM만 Slowdown.

원인 가설: GaussianHMM이 표준화 데이터(분산~1)에서 `min_covar=1e-3`로 학습돼 방출
분포가 뾰족해지고 사후확률이 0/100으로 **포화**된다. blend=0.6·HMM+0.4·RF라, 포화된
HMM이 분산된 RF를 압도 → 단기 변동성 블립이 비중을 좌우(휘프소).

가설적 처방 (A): 공분산 floor를 올려 방출을 넓히면 여러 상태가 확률을 분담 →
단일 HMM의 노이즈성 플립 영향이 줄어든다.

## 해석 — 처방이 가설대로 작동하지 않음

- **회전이 줄지 않는다 (핵심 반증)**: floor를 올려도 리밸 횟수가 오히려 증가(542→545~546),
  tx누적도 개선 없음. "포화 완화 → 휘프소 감소" 가설이 백테스트에서 성립하지 않음.
- **위험조정 성과 하락**: floor≥0.05에서 Sharpe 0.84→0.79~0.82, MaxDD -9.4→-9.7%로 열위.
- **위기 방어 약화**: Bear22 낙폭이 floor↑에서 -8.5→-8.9%로 깊어짐.
- 0.01은 baseline과 사실상 동일(노이즈 범위) — 의미 있는 개선 아님.

왜? HMM 포화의 다운스트림 영향은 이미 (1) blend EWMA 평활, (2) RF 블렌딩(0.4),
(3) confidence_smoothing, (4) confirmation 히스테리시스가 흡수하고 있다. 여기서 HMM을
더 흐릿하게 만들면 결정력만 잃어 작은 플립이 더 잦아지고(회전↑) 위기 진입이 둔해진다
(Bear22 악화). 즉 2010-2025 전구간에서 포화된 HMM은 대체로 성과 중립이며, 누그러뜨리면
손해다. 최근 휘프소는 전구간 알파 누수가 아니라 국소 현상에 가깝다.

## 결론 및 후속

- **min_covar는 baseline 0.001 유지** (config 기본값 변경 없음). 코드는 config로 토글
  가능하게 배선만 남겨둠(기본값에서 동작 동일).
- 최근 휘프소 완화는 이미 적용·검증된 [[confidence_smoothing]](experiment_2026-06-08_confidence_smoothing.md)이
  담당. 추가로 검토할 후속 레버: (B) rf_weight↑(포화 HMM 견제), (C) 비-Crisis 디리스크
  코로보레이션 게이팅. 둘 다 별도 백테스트 검증 필요.
