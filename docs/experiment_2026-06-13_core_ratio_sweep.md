# 실험: 코어(고정 Goldilocks) 비중 스윕 0/30/50/70% — 현행 시스템 + 고정 4지표

> **요약**: ① core+satellite의 코어(정적 Goldilocks·vol 면제 회복 앵커) 비중을 0(코어 없음)·30(현행)·50·70%로 스윕해 4지표가 어떻게 움직이는지 측정했다(vol ON floor 0.65, drift 리밸, DD scaling OFF 고정 — 규칙5 합의). ② **Martin이 30%에서 2.54로 최고점인 역U자**가 나왔다: core 0%는 MaxDD -11.0%·Ulcer 2.93으로 *깊이*는 가장 얕지만 최장 underwater 644일·Martin 2.36·CAGR 10.9%로 회복이 느리고 효율이 낮고, core 50/70%는 정적 Goldilocks 질량이 커져 MaxDD -17.5→-20.0%·Martin 2.36→2.26으로 방어가 무너진다. 30%만 회복일 82일·최장UW 409일로 가장 짧다. ③ **현행 core 30% 유지** — 4지표 1차(Martin·회복기간)에서 단독 최고점이라 변경 근거 없음. 코어를 늘리면 수익(CAGR↑)·낙폭 깊이(완화)는 한 방향으로 움직이나 Martin·회복기간은 30%를 정점으로 양쪽이 나빠진다.

- 날짜: 2026-06-13
- 코드: `scripts/sweep_core_ratio.py` (라이브/엔진/config 변경 없음 — 진단)
- 기간: 2010-01-01 ~ 2025-04-30, USD 단일통화, 현행 config 로드
- 고정(규칙5 합의): vol ON(floor 0.65), drift 리밸(config 0.015), DD scaling OFF, 나머지 현행. core_ratio만 토글.

## 동기

core 30%는 `experiment_2026-06-12_core_satellite`에서 채택됐으나 그때와 floor·평가 환경이 달라졌다
(floor 0.80→0.65, drift 모드 확정). 충돌 피처쌍 분석(#2: core30이 vol/gate/blend를 70%로 희석,

vol 면제·정적 Goldilocks 질량↑ = 회복 참여↑·엔진 방어 반응↓.

## 해석

### 판정 — Martin 역U자, 30%가 단독 정점
Martin: 2.36(0%) → **2.54(30%)** → 2.36(50%) → 2.26(70%). 30%가 양옆을 0.18 이상 앞서는
명확한 정점이다. 회복일도 30%가 82일로 최단, 최장UW도 409일로 최단. 4지표 1차(Martin·회복기간)
전부 30% 우위.

### 두 방향의 단조 변화 (역U자의 정체)
- **CAGR·낙폭 깊이는 코어와 단조**: 코어↑ → CAGR 10.9→14.3%(↑), MaxDD -11.0→-20.0%(깊어짐),
  Ulcer 2.93→4.56(↑). 정적 Goldilocks는 equity 중심이라 코어를 늘릴수록 평시 수익은 커지되
  방어가 사라진다.
- **회복기간은 양 끝에서 동시에 악화**: core 0%는 깊이는 얕으나(-11%) 최장UW 644일로 *가장 길다* —
  순수 엔진(코어 없음)은 깊게 안 빠지지만 vol targeting이 equity를 오래 눌러 회복 참여가 약해
  물려있는 기간이 길다(깊이↔길이 트레이드오프). core 70%는 너무 깊이 빠져(-20%) 회복도 568일로 길다.
  **30%만 "적당히 방어 + 회복 앵커 참여"의 균형점**이라 UW 409일로 가장 짧다.

### core 0%의 함정 — 얕은 MaxDD에 속지 말 것
core 0%는 MaxDD -11.0%·Ulcer 2.93으로 *깊이* 지표만 보면 최고다. 그러나 4지표 1차인 Martin
(2.36, 최저 수준)·회복기간(UW 644일, 최장)에서는 꼴찌급이다. CAGR도 10.9%로 가장 낮다. 깊이만
보고 코어를 없애면 "얕지만 길고 수익 낮은" 포트폴리오가 된다 — 규칙4가 경계하는 단일지표 함정.

## 결론

- **현행 core 30% 유지. config 변경 없음.** 4지표 1차(Martin 2.54·회복일 82·최장UW 409) 단독 정점.
- 이 스윕은 floor 0.65·drift 환경에서도 core30 채택이 유효함을 재확인한다(이전 채택은 floor 0.80
  환경이었음). core 비중은 *값*을 더 튜닝할 게 아니라 30%가 회복기간 균형점이라는 구조적 결론.
- 보조: 만약 우선순위가 *낙폭 깊이 단독*이면 core 0%(-11%)가 유리하나, 그 대가가 회복 644일·
  Martin 최저라 본 포트폴리오 목표(하락 회피 = 깊이+길이, 3~5년 보유)와 어긋난다.
