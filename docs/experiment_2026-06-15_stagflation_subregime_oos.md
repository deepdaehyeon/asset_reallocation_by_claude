# 스태그 하위국면 분기 OOS — 진단은 옳으나 비중 처방은 엔진 흡수 (라이브 미채택)

## SUMMARY
- **무엇을**: 진단(`stagflation_realrate_split.py`)이 확정한 "실질금리 3개월 변화가 스태그 하위국면을 가른다"를 실제 비중 처방으로 옮겨 OOS 검증. engine.py에 되돌림 가능한 `stagflation_subregime` 오버레이(긴축형=실질금리↑일 때 스태그 슬롯을 tightening_targets로 교체)를 넣고, 긴축형에서만 금·tips를 채권/cash로 옮기는 2변형(ST1 금·tips만, ST2 +commodity)을 4지표 워크포워드 OOS로 검증.
- **핵심 수치**: ST1(금18→10·tips8→4→bond/cash): 학습 ΔMartin −0.03 / 검증 −0.03, 회복기간 307→342 악화 → 흡수~소폭 손해. ST2(+comm18→14): 학습 +0.03 / 검증 −0.14, 회복 409→556 악화 → 과적합 지문. 디스인플레형은 base 유지(단일변수 비교).
- **채택 여부·결론**: **라이브 미채택, 스태그 현행 유지.** 진단 자체는 견고(실질금리 방향이 2010s↔2022 방어자산 역전의 원인 변수)하나, 비중 처방은 엔진(스태그 저블렌드·core30 70% 희석·vol targeting 0.08 흡수)을 못 이긴다. 오버레이는 코드에 보존하되 비활성(knob off). per-regime 비중 무력의 재확인([[feedback-regime-targets-no-tuning]]) — 다만 이번엔 "비중조정"이 아니라 "구조 분기"도 같은 운명임을 보인 데 의미.

## 배경
`stagflation_realrate_split.py`에서 실질금리 3개월 변화가 스태그를 깨끗이 가름을 확인(하락 시
금M+25.25·tips+22.71·bond+77.11 영웅, 상승 시 금−0.96·tips−0.24 역전). 외부 리뷰의 "실질금리가
2022의 열쇠" 가설이 진단 통과 → 구조(하위국면 분기)로 개선되는지 비중 처방으로 검증.

## 구현 (되돌림 가능 오버레이)
- `backtest/engine.py` `_subregime_config(as_of)`: `stagflation_subregime.enabled`이고 as_of의
  `real_rate_chg_3m ≥ threshold(0)`이면 `regime_targets["Stagflation"]`만 `tightening_targets`로
  교체한 config를 blend에 전달. 비활성/디스인플레형(실질금리↓)이면 self.config 그대로(동작 불변).
- `_get_regime`이 리밸 직전 `self._current_as_of` 저장 → 호출부 5곳 미변경. baseline은 키 부재라
  완전 동일(회귀 0).

- ST2: ST1 + commodity 0.18→0.14 → cash 추가 +0.04 (합 1.00)

## 해석·결론
- **진단 ≠ 처방.** 실질금리가 두 스태그를 가른다는 동시점 진단은 견고하나, 그 차이를 *고정 비중*
  으로 옮기면 엔진이 흡수한다: ① 스태그는 드문 레짐이라 blend 비중이 작고, ② core30이 30%를
  Goldilocks로 고정해 변경을 70%로 희석, ③ vol targeting이 스태그 목표 0.08로 이미 매우 방어적
  이라 "금→cash"가 최종 포트폴리오 변동성에 거의 안 잡힘. 2022 긴축 스태그는 엔진이 *이미*
  방어 모드라 추가 비중 처방의 한계효용이 0에 수렴.
- **per-regime 무력의 일반화**: 지금까지 비중조정 후보는 10건째 OOS 기각(C1/C2/G/R/S1~S3/CR1/CR2
  + ST1/ST2). SL2만 통과. 이번 결과의 새로움 = "비중 미세조정"뿐 아니라 "실질금리 기반 구조 분기"
  도 같은 흡수 운명임을 보임. 즉 스태그 개선의 병목은 *분기축 발견*이 아니라 *엔진이 비중차를
  흡수하는 구조* 자체.
- **그럼 어디서 개선되나**: 비중·분기가 아니라 (가) vol targeting 파라미터(스태그 target 0.08
  자체)·(나) 진입 적시성([[feedback-regime-timing-lever]])·(다) core30이 희석하는 정도. 실질금리는
  비중 스위치보다 **vol target을 긴축형 스태그에서 더 조이는** 식으로 쓰는 게 효과 가능성 높음(미검증).
