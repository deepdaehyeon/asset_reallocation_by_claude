# 실험: blend_probs EWMA 평활 (외부 비평 #6-c whipsaw 억제)

> **요약**: ① blend_probs에 EWMA 평활(α=0~0.9 스윕)을 적용해 레짐 전환의 whipsaw 57.6%를 줄이는 실험을 수행했다. ② α=0.5에서 whipsaw 57.6%→47.3%(-10.3pp), 전환 횟수 243→182회, Sharpe 영향 -0.013(동등 수준), MaxDD -1.18pp 악화로 허용 범위 내임을 확인했다. ③ α=0.5 채택됨 — config `regime_filter.blend_smoothing_alpha: 0.5`로 적용됐으며, α 상승 시 위기 진입 지연 부작용이 있어 더 높은 값은 기각됐다.

*작성일: 2026-05-27*
*상태: 비교 완료. **α=0.5 채택 권장** (whipsaw 57.6%→47.3%, 성과 영향 미미).*

## 배경

이전 진단(`docs/experiment_2026-05-27_regime_observability.md`)에서:
- 레짐 전환의 **57.6%가 21일 내 직전 레짐으로 복귀** (whipsaw).
- 라이브의 `RegimeFilter` confirmation/cooldown은 표시·알림만 영향 — config 주석에도 "실제 비중은 연속 블렌딩 적용" 명시.
- 따라서 **실제 매매에 미치는 whipsaw**를 줄이려면 `blend_probs` 자체를 평활해야 함.

## 구현

EWMA 평활:
```
new_blend = α · prev_blend + (1-α) · raw_blend
```
- 정규화 후 사용.
- `regime_filter.blend_smoothing_alpha` config (기본 `0.0` = 평활 없음, 호환).
- `backtest/engine.py`: `_prev_blend` 인스턴스 상태로 워크포워드 사이 유지.
- `trading/run.py`: `state.json`의 `prev_blend_probs`로 일별 호출 간 영속화.

## 해석

1. **α=0.5가 가장 균형 잡힌 선택**.
   - whipsaw 10.3pp 감소(57.6→47.3%): 단기 전환 노이즈가 의미 있게 줄어듦.
   - Sharpe -0.013: 실질적으로 동등.
   - MaxDD -1.18pp: 보수적 허용 범위(-2pp 이내).
   - 전환 횟수 243→182회: 매매 빈도 25% 감소 → 거래비용 절감 효과도 있음.

2. **α를 더 올리면(0.7+) MaxDD 악화 폭이 커짐**.
   - 평활이 너무 강해 위기 진입 시그널이 약화. 진단에서 본 "위험 진입 전환이 후행적"인 패턴이 더 심해짐.
   - 흥미롭게 α=0.9는 Sharpe가 살짝 오르지만 MaxDD -4.25%로 큰 폭 악화 → MaxDD 보호가 더 중요한 운영 상황에서는 부적합.

3. **Crisis 일수 감소(-49일, -23%)는 부작용**.
   - 평활이 일시적 Crisis 신호를 흡수해 진입을 지연/회피시킴.
   - 다만 baseline의 Crisis 일수 자체에 후행적 진입 문제(이전 진단의 평균 수익 +0.38% 양수)가 있었으므로, "진정한 위기"를 덜 잡는다고 단언하기 어려움.
   - MaxDD 악화가 1.18pp로 제한적이라 실질 위험은 통제 가능 수준.

## 적용 방법

```yaml
regime_filter:
  blend_smoothing_alpha: 0.5
```

코드 변경 없이 config만으로 즉시 적용·롤백 가능.

## 한계

- 평활이 Crisis 진입을 지연시키는 효과는 부작용. 후속 작업으로 **위기 시그널은 평활 우회**하는 비대칭 평활 검토 가능 (예: `realized_vol` 또는 `vix`가 임계 초과 시 α=0으로 강제 fallback).
- α=0.5도 여전히 whipsaw 47.3%로 절반 가까이 — 분류 신호 자체의 노이즈가 본질적 원인. 더 근본적 개선은 `detect_regime` 임계 재검토(다음 작업).
