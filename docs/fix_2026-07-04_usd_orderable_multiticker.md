# USD 주문가능금액 조회 — 단일 종목 호가 실패가 계좌 전체 예산을 삭감하던 버그 수정

> **요약**: ① 라이브 회전율이 백테스트의 ~10배(6월 94%/월, 07-03 하루 21%)인 근본 원인을 로그로 추적한 결과, `_fetch_usd_orderable`가 USD 매수목록의 **첫 종목 하나(예: VEA)** 호가로만 계좌 주문가능금액을 조회하고, 그 종목 호가가 0(레이트리밋·미장 마감 무렵 stale)이면 함수 전체가 예외로 떨어져 매도 *전* 현금 폴백으로 **USD 예산을 16.8~29%까지 삭감**하는 버그를 발견했다. ② 삭감된 예산 탓에 USD가 목표에 도달하지 못하고(VEA 목표3.3%/현재1.2% 이틀째 고착) 총 drift가 ~12%에 붙박이면, KRW·USD 공유 트리거(`_compute_side_drifts`)가 자기목표에 이미 붙어있는 KRW까지 매일 강제 리밸(`강제 평준화`)시켜 회전이 눈덩이처럼 불어났다. ③ `ovrs_ord_psbl_amt`가 계좌 단위 값이라 유효 호가 하나면 되는 점을 이용해, 한 종목 호가 실패 시 **다음 후보 종목 호가로 순차 재시도**하고 전 종목 실패 시에만 현금 폴백하도록 수정(모의 3케이스 통과). 백테스트는 깨끗한 과거종가·완전체결이라 이 층 자체가 없어 못 보던 라이브 전용 마찰이며, 이 수정으로 라이브가 백테스트에 수렴할 것으로 기대.

## 근본 원인 (로그 증거, 2026-06/07)

USD 리밸(04~05시 KST = 미장 마감 무렵) 실행 중:
```
WARNING: API 호출 횟수를 초과하였습니다.          ← KIS 레이트리밋
[경고] VEA ask 가격 비정상: 0.0                  ← 호가 0 조회
[주문가능 cap] USD: 10,790,776원 → 1,815,386원 (16.8%)   ← 예산 삭감
[skip] VEA: 가격 조회 실패
```
예산 삭감 실측: 06/24 →18.6%, 06/25 →16.8%, 07/04 →29.4%.

### 인과 사슬
1. USD 매수목록 첫 종목(`ref_ticker = buys[0][0]`)이 VEA/DBC면 호가가 0으로 자주 실패.
2. `_fetch_usd_orderable`가 예외 → `_usd_cash_krw(매도 전 출금가능현금)×0.98` 폴백 반환.
   - 당일 매도대금(KIS는 미결제 매도대금도 매수한도에 반영)을 무시해 실제보다 훨씬 작음.
3. 그 폴백으로 전체 USD 매수를 scale down → USD가 목표 미달로 굳음.
4. 총 drift(merge 기준)가 ~12%에 고착.
5. `_compute_side_drifts`가 총합 단일 drift를 KRW/USD 양쪽에 공유(`return drift, drift`)
   → USD가 못 맞으면 자기목표에 이미 붙은 KRW도 매일 강제 트리거.
6. `강제 평준화`(per_ticker_drift_threshold=0)가 0.3~0.8% 미세차까지 전량 주문 → KRW 무고한 churn.
   → 6월 총 체결 2.07억 = NAV의 94%/월.

## 수정 (trading/executor.py)

- `_fetch_usd_orderable(acc_name, ref_ticker: str)` → `(acc_name, ref_tickers)` (str 또는 list).
- 후보 종목을 순차 시도해 **유효 호가를 주는 첫 종목**으로 계좌 단위 `ovrs_ord_psbl_amt` 조회.
  한 종목 실패는 경고 후 다음 종목으로 continue. **전 종목 실패 시에만** 현금 폴백.
- 호출부(pre-cap): `ref_ticker = buys[0][0]` → `ref_tickers = [t for t, _ in buys]` (전 매수 후보 전달).
- 재시도 호출부(`_execute_buy_capped`, 특정 종목)는 str 그대로 넘겨 하위호환(내부 list 정규화).

## 왜 백테스트는 못 보나

깨끗한 과거 종가(호가0 없음)·완전체결·리밸 후 `drift=0` 리셋이라 가격피드·API·계좌분리 층 자체가
백테스트에 없음. → 이 수정은 라이브를 백테스트 가정에 근접시키는 방향.
