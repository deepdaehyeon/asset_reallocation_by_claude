# 실험: 평활 2겹 + 위험 진입 타이밍 통합 격자 — 현행 시스템 + 고정 4지표

> **요약**: ① 충돌 피처쌍 분석 #3(평활 2겹: blend_smoothing_alpha EWMA 평활 ↔ confidence_smoothing 신뢰도 가변 평활)·#4(느린 평활 ↔ regime_timing_source=rule 빠른 위험 진입, 방향이 반대라 상쇄되는지)를 하나의 격자(평활 3레벨 × timing 2 = 6셀)로 검증했다. 구조적 사실(engine.py:250,262: confidence_smoothing은 blend_alpha>0일 때만 작동)에 맞춰 평활을 {없음(α0)/고정(α0.5)/가변(α0.5+cs=현행)} 3레벨로 구성. ② **두 평활 모두 제값을 한다**: rule 기준 Martin 2.29(없음)→2.47(고정)→**2.54(가변)**로 단조 개선, 동시에 회전 tx 4.82→3.64→3.14%로 단조 감소; 특히 가변평활(confidence_smoothing) 레이어가 최장 underwater를 521→409일로 크게 줄이는 게 고유 기여다. **빠른 진입(rule)은 평활과 상쇄되지 않고** 모든 평활 레벨에서 ensemble을 Martin +0.05~0.08 앞선다(blend 비중=HMM 평활, timing=vol 티어·트리거로 작용 채널이 달라 충돌 아님). ③ **현행(rule + 가변평활) 유지** — 6셀 중 Martin 2.54 최고 + 최장UW 409일 최단 동시 달성. 평활을 끄면 회전이 +1.68%p 폭증하고 UW가 409→523일로 길어진다. config 변경 없음.

- 날짜: 2026-06-13

- 기간: 2010-01-01 ~ 2025-04-30, USD 단일통화, 현행 config 로드

## 동기

충돌 피처쌍 분석에서 사용자가 #3·#4를 개별 실험 대상으로 지목했다.
- **#3 평활 2겹**: `blend_smoothing_alpha`(blend_probs EWMA 평활)와 `confidence_smoothing`
  (신뢰도 낮으면 채택 지연)이 같은 평활 파이프라인을 공유해 어느 게 진짜 효과인지 불명.
- **#4 평활 ↔ 빠른 진입**: `regime_timing_source=rule`은 앙상블보다 +3~5d 빠른 위험 진입
  (천천히 가라는 평활과 *반대 방향*) — 상쇄되어 순효과가 사라지는지.

### 구조적 사실 (격자 설계 근거)
engine.py:250·262 — confidence_smoothing은 `blend_smoothing_alpha > 0`일 때만 작동한다. 즉
두 평활은 독립이 아니라 **confidence_smoothing이 고정 평활 위에 얹히는 변조**다. 따라서 평활을
{없음(α0) / 고정(α0.5·cs off) / 가변(α0.5·cs on=현행)} 3레벨 단조 사다리로 구성하는 게 옳다.

## 해석

### #3 — 두 평활 레이어 모두 제값을 한다 (단조 개선)
rule 타이밍 고정 시 평활을 더할수록 4지표 1차가 단조 개선:
- **Martin**: 2.29(없음) → 2.47(고정, +0.18) → **2.54(가변, +0.07)**.
- **회전(tx)**: 4.82% → 3.64% → 3.14% — 평활이 whipsaw 매매를 단조로 줄인다(가장 큰 실용 효과).
- **고정평활(blend_alpha)의 기여**: Martin +0.18, tx -1.18%p. whipsaw 억제로 효율↑.
- **가변평활(confidence_smoothing)의 고유 기여**: 고정평활 위에서 **최장UW 521→409일**(-112일)
  + tx 3.64→3.14% + Martin +0.07. 신뢰도 낮을 때 채택을 더 늦춰 가짜 플립을 추가 억제 →
  물려있는 기간을 줄인다. **두 레이어는 중복이 아니라 역할이 다르다**(고정=회전·효율, 가변=UW 단축).

### #4 — 빠른 진입은 평활과 상쇄되지 않는다 (충돌 가설 기각)
rule vs ensemble을 평활 레벨별로 비교하면 rule이 전 구간 우위:
- α0: 2.29 vs 2.21 (+0.08) / α0.5: 2.47 vs 2.43 (+0.04) / α0.5cs: 2.54 vs 2.49 (+0.05).

평활이 "천천히 바꿔라"여도 rule 타이밍은 **작용 채널이 달라** 효과가 죽지 않는다: blend(비중)는
**항상 HMM 평활**을 거치고, regime_timing_source는 **vol 티어·리밸 트리거**의 acting regime만
rule로 빠르게 잡는다(engine.py:276). 즉 "느린 비중 + 빠른 위험 인식"이 공존하며 서로 다른
레버라 상쇄되지 않는다. #4의 "방향이 반대라 상쇄" 가설은 기각.

### 깊이 ↔ 길이 (보조 관찰)
평활없음(α0)은 MaxDD -13.9%로 *깊이*는 가장 얕으나 최장UW 523일·Martin 최저·tx 최고다.
평활은 MaxDD를 약 1%p 깊게 하는 대신 UW를 409일로 줄이고 회전을 크게 낮춘다 — 규칙4(회복기간·
Martin 1차) 기준 명백한 순이득. 깊이만 보는 단일지표 함정에 빠지지 말 것.

## 결론

- **현행(rule + 가변평활 α0.5·cs) 유지. config 변경 없음.** 6셀 중 Martin 2.54 최고 +
  최장UW 409일 최단을 동시 달성하는 유일 셀.
- #3 답: 두 평활은 중복이 아니다 — 고정평활은 회전·효율, 가변평활(confidence_smoothing)은
  underwater 단축이 고유 역할. 둘 다 유지가 정당.
- #4 답: 빠른 위험 진입(rule)과 평활은 충돌하지 않는다 — 작용 채널(비중 vs vol 티어/트리거)이
  분리돼 있어 각각 독립적으로 기여. 기존 rule-timing 채택(2026-05-31) 재확인.
