# 실험: RF / HMM 비율 스윕 (blend = (1-w)·HMM + w·RF) (2026-06-17)

## 요약 (3문장)
1. **무엇을** — 위성 비중을 만드는 blend의 HMM(비지도)/RF(룰을 베낀 지도) 혼합비 `rf_weight`를 0.0(순수 HMM)~1.0(순수 RF)으로 스윕해 고정 4지표로 비교했다. core30 OFF(순수 신호)·평활 ON·drift·tx·2010~2025.
2. **핵심 수치** — **순수 HMM(RF0)이 명백히 최악**(Ulcer 3.79, 회복 **359일**, Martin 1.65). RF 비중을 올릴수록 Martin 1.65→1.97·Ulcer 3.79→3.27·회복 359→110일·롤3y최악 3.8→4.5%로 거의 단조 개선. 스윗스팟은 **RF 60~80%**(현행 RF40은 Martin 1.74로 중하위).
3. **결론** — 방향성은 견고: **RF를 더 실으면(=룰에 더 가깝게·덜 흔들리게) 4지표가 개선**된다. 다만 in-sample 단일기간이라 **라이브 변경 전 OOS 검증 필요** — 우선 `rf_weight`를 0.40→**0.60~0.70**으로 올리는 안을 후보로 보류(자동 채택 안 함, [[feedback-regime-targets-no-tuning]] 취지로 한 점 최고값보다 구간 안정성으로 판단).

## 배경
- 비중 blend = (1-w)·HMM_probs + w·RF_probs (`run.py:360`·`engine.py:245`). HMM은 비지도(매일 재학습, 흔들림), RF는 룰 레짐을 정답지로 학습한 지도 분류기(forward_window=0 → 오늘 룰을 베낌). 즉 **w↑ = "더 룰스럽고 안정적", w↓ = "더 HMM답고 흔들림"**.
- 범위 합의(사용자): core30 OFF로 희석 제거, 평활 ON(현행). 재현 `scripts/rf_hmm_weight_sweep.py`.

## 해석
- **순수 HMM의 비지도 흔들림이 성능을 해친다**: RF0의 회복 359일·Martin 1.65는 전 구간 최악. 이번 세션 내내 진단한 "HMM 일별 churn이 해롭다"와 정확히 일치 — RF(룰 기반·안정)를 섞을수록 좋아진다.
- **4지표 일관 개선**: RF 비중↑에 따라 Martin↑·Ulcer↓·회복↓·롤3y최악↑·롤3y중앙↑이 거의 단조. 한 지표만 반대인 건 롤5y최악(현행 60/40이 4.6%로 최저) 정도라, 노이즈가 아니라 방향성 신호로 본다.
- **스윗스팟 RF60~80%**: Martin 1.94~1.97, Ulcer 3.27~3.40, 회복 110~124. 순수 RF(RF100)는 HMM 독립신호를 완전히 버리고 MaxDD -12.7%로 약간 악화 → RF60~80이 균형.
- **tx**: 전 구간 3.6~3.8%로 유사(RF-heavy가 회전을 늘리지 않음). 단 백테스트 tx는 라이브 회전을 과소반영([[project-live-turnover-vs-backtest-gap]]).

## 결론·후속
- 방향: **rf_weight 0.40 → 0.60~0.70 상향이 4지표상 유리**. 단 in-sample 단일기간 — 채택 전 서브기간/OOS 견고성 확인 권장(스태그·완화형 등 레짐 편중 구간에서 RF가 룰을 과적합하는지 점검).
- 본 실험은 core30 OFF 순수신호 기준. 라이브(core30 ON)에선 효과가 위성 70%로 희석되므로 라이브 적용 시 체감 차이는 더 작다.
- 자동 채택하지 않음 — 사용자 확인 후 결정.
