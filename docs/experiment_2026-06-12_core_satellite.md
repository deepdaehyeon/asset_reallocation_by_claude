# 실험: core+satellite 구조 + 레짐 블렌드 ON/OFF

- 날짜: 2026-06-12
- 코드: `scripts/experiment_core_satellite.py`
- 사용자 제안: core(Goldilocks 고정) + satellite(레짐 스위칭) 구조.
- 사용자 정정: 요즘 매매는 레짐 블렌딩(0.6×HMM+0.4×RF+EWMA)이 confirmed 레짐 고정
  상태에서도 목표 비중을 매일 밀어내서 생긴다. 블렌드 없이 홀드하면 리밸런싱 효과만 남는다.
- 질문 답: **백테스트는 레짐 블렌딩을 한다**(`_target_weights`→`blend_regime_targets`).
  그래서 블렌드 ON/OFF를 함께 비교했다.

## 변형 (2010-01-01~2025-04-30, USD 단일통화)

1. baseline(블렌드ON): 현행 — 스위칭 + 블렌드 + vol타겟
2. 하드레짐(블렌드OFF): 스위칭은 하되 blend를 argmax 1-hot → 레짐 바뀔 때만 계단식 변경
3. core50+sat: 50% Goldilocks 고정(vol·blend 없음) + 50% satellite(현행 엔진)
4. core70+sat: 70% core
5. 순수홀드(Goldilocks): blend={Goldilocks:1}, vol OFF

## 해석

### 1. 블렌드는 churn을 *만드는* 게 아니라 *줄인다* — 직관과 반대

블렌드 OFF(하드레짐)의 tx누적이 **7.22%로 ON(3.84%)의 약 2배**다. 리밸 *횟수*는 오히려
적은데(485 vs 541) 거래 *규모*가 크다. 이유: 블렌드가 없으면 레짐이 바뀔 때마다 책 전체가
한 번에 점프(계단식)하지만, 블렌드는 그 전환을 여러 날에 걸쳐 매끄럽게 나눠 whipsaw를
줄인다. 즉 **블렌드는 전환 평활 장치**이지 과회전 원인이 아니다([[feedback-agility-over-turnover-reduction]] 정합).
→ "블렌드 빼면 거래 줄어든다"는 백테스트에서 **반증**. 거래를 진짜 줄이는 건 블렌드 제거가
아니라 **스위칭 자체를 줄이는 것**(순수홀드 tx 1.24%가 최저).

### 2. 그런데 사용자의 라이브 관찰은 여전히 맞다 — 원인이 다를 뿐

백테스트는 drift threshold(1.5%)로 작은 목표 이동을 흡수해 15년 541회(≈주1회) 리밸에 그친다.
**라이브는 `강제 평준화`로 그 drift 데드밴드를 무시**(`per_ticker_drift_threshold 무시 —
모든 차이 주문 생성`)하기 때문에, 블렌드의 매일 미세 이동이 그대로 매일 주문이 된다.
즉 라이브 일일 churn = 블렌드 미세드리프트 × **데드밴드 우회** + USD합성 잔차
([[project-live-turnover-vs-backtest-gap]]). 블렌드를 빼도 이건 안 고쳐진다 — 데드밴드 복원과
합성 제거가 직접 약이다.

### 3. core+satellite는 좋은 구조다 — 단, 이유는 '블렌드 제거'가 아니라 '고정 코어'

- **churn 감소:** baseline 3.84% → core50 2.51% → core70 1.87%. core 부분은 목표가
  Goldilocks로 고정이라 블렌드·강제평준화가 와도 거래를 안 만든다. 라이브에서도 core 비중만큼
  일일 churn이 줄어든다(스크린을 잠재우는 직접 효과).
- **수익·방어 다이얼:** core 비중이 커질수록 CAGR↑·MaxDD↓ 단조. core50은 baseline 대비
  CAGR +3.2%p(13.3%) 대가로 MaxDD -7%p(-16.9%).
- **결정적: core50이 Martin(Ulcer 조정 수익)에서 전 변형 중 최고(2.39).** baseline 2.13,
  순수홀드 2.07보다 높다. satellite의 방어가 Ulcer 상승을 거의 막는 사이(3.93→3.91) CAGR만
  뛰어서, 50/50은 단순 중간점이 아니라 **위험조정 효율의 스윗스팟**이다.

### 4. 하드레짐(블렌드 OFF)은 전부 열위 — 채택 금지

tx 2배, MaxDD 더 깊고(-13.2%), Martin 최저(1.99). 블렌드 제거는 어느 축에서도 이득이 없다.

## 결론

- **블렌드는 유지**한다(churn↓·MaxDD↓·whipsaw 평활). 제거는 반증됨.
- 라이브 일일 churn의 진짜 원인은 **강제평준화(데드밴드 우회) + USD합성**이지 블렌드가 아니다.
- **core+satellite(특히 core50)는 채택 후보로 강함:** 라이브 churn을 core 비중만큼 줄이고,
  수익을 +3%p 올리며, Martin 기준 위험조정 효율이 오히려 최고. 대가는 MaxDD -9.9%→-16.9%
  (장기보유자에겐 패닉 이탈 방지값만 약화, [[experiment-2026-06-12-holder-metrics]] 참고).

## 한계

- 백테스트는 USD 단일통화·단일 포트폴리오 — 라이브 합성 순환매 미반영. core+satellite의
  실제 churn 감소폭은 라이브 회전율 재측정으로 검증 필요.
- core의 Goldilocks 자산은 대부분 USD(DBC·DBMF·factor)라, '고정 코어'도 USD합성/환전은
  따라온다 — core를 진짜 hands-off로 만들려면 KRW-native화나 USD 선충전이 선행돼야 함.
- 음수% 0%(롤링)는 2010~2025 상승 우위 표본 산물 — 일반화 금지.
- 실험 구현은 클래스레벨 블렌드(core=Goldilocks vol-off, satellite=현행). 실제 운용 구현 시
  계좌·회계 분리 방식에 따라 결과가 달라질 수 있음 — 이건 구조 타당성 1차 측정.
